What I want this project to become:
 My goal is to build a secure, self-hosted music streaming application that provides an experience similar to Spotify while allowing me to maintain complete ownership 
 and control of my music library.
 Long-term, I would like the application to be reliable and secure enough that my friends could use it as an alternative for sharing and streaming music.

Why am I Building it:
  As I begin my career in cybersecurity through a DevSecOps internship, I want to strengthen my understanding of how software is designed, deployed, and secured throughout its lifecycle. 
  Rather than only studying concepts, 
  I want to build a real application that allows me to apply DevSecOps practices, document my decisions, and gain practical experience with modern development workflows.

What you hope to learn:
 Through this project, I hope to gain a deeper understanding of:

  Docker and containerized applications.
  Linux system administration.
  Database architecture and PostgreSQL.
  Secure authentication and authorization.
  Networking fundamentals, including reverse proxies, HTTPS, and DNS.
  CI/CD pipelines and automation.
  Monitoring, logging, and incident response.
  Secure software design and the tradeoffs organizations make between security, usability, and cost.

  Ultimately, I want to understand not only how applications are built, but also why certain architectural and security decisions are made in production environments.

  Engineering Principles:
  
  Security should be considered from the beginning, not added later.
  Every architectural decision should have a documented rationale.
  Infrastructure should be reproducible using code and configuration.
  The system should follow the principle of least privilege wherever practical.
  Documentation should evolve alongside the application.
  The project should prioritize learning and understanding over unnecessary complexity.
